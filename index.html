Introduction

The "web" is made up of several parts. People use their browsers to view web pages. These pages are stored on other computers called servers that "serve" information to client computers through the internet using the http protocol. This information can take many forms, but the most common are HTML documents. HTML is a language used to format the data displayed on webpages.

Working with HTML

HTML functions by using tags to add special properties to the text we write. Tags can make text bold, create spacing, and even embed videos, pictures, and links to other web sites on our page. *Note: Computers will not forgive errors in syntax*

Basic tags

  • <b> bold
  • <em> italics
  • <br> line break
  • <p> paragraph

Some tags like anchor include attributes that modify the function performed. Some tags like break are void tags and do not require a closing tag. Some tags like paragraph create a block around their content grouping it and seperating from other text.

CSS Usage

CSS takes our HTML code to another level. With CSS we add style on top of the structure provided by our HTML. Create a seperate file .css, and link to that in the head of the HTML document. CSS works similarly to writing HTML. It consists of selectors that choose the element to apply style to and rules between {}.Rules consist of attributes like background-color, and values which in this case would be the color we wanted to use stated: background-color: blue;. Selectors can specify targets by element type (like paragraphs or divs), or by classes. In the Html file we can declare class names for different elements to plan out how to apply styles.The C is CSS stands for Cascading, this refers to the inheritance structure of CSS. This means that styles applied to an element will be inherited by nested elements within. We can create additional more specific classes, for instance if we had three paragraphs, and wanted one to be a different style than the others.

More Python

This session introduced some more hardcore programming concepts, mainly variables and strings. Variables allow us to save some data for use. Strings give us a way to work with data other than numbers.

Variables

A variable is a place to assign a value and save it to memory. We assign a value to a variable by writing a line of code naming the variable and assigning a value "example=1". Now whenever we call example in later code, the interpreter is going to know to use the integer 1. In programming we often use an equal sign to assign a value, in HTML this was done for naming classes.

Strings

A string is simply a sequence of characters. By using quotes we can work with non-numerical data. If we enter 2+2 in python we get 4. If we enter "2"+"2" we get 22. This is because designating the 2 a string using quotation marks tells the interpreter to treat the 2 as a character, and not an integer to perform arithmetic on

Session 7

Equality Comparisons

We can use code to make different decisions based on the value of data. To do this we use comparisons. Less than, greater than, "==" which is equal, and "!=" or not equal all evaluate to a Boolean True or False. Using this we can make our code branch out depending on the data.

If and Else

If statements are what we use to make these branching decisions in our code. The syntax is:

If (test expression(such as x == 1)):
Block of code

Our program will run, then it will evaluate the if statement(running the block of code if it meets the test conditions) and then move on to the next instructions. We can use an Else statement, which follows the same syntax as If, to run another block of code when data does not meet the test conditions.

While loops

Using While we can execute a conditional block of code an unlimited amount of times as long as the test condition remains true. While uses the same syntax as the other conditional statements. We can use break, to stop the while loop, by using another condition. That looks like this

while (test expression):
Block of code
if (test expression):
Break

Lists

Lists are a type of structure data similar to, but different from, strings. Strings are a sequence of characters but lists can contain anything; strings, numbers, even other lists. We use lists like this.

Working with Lists

Lists unlike strings can be changed. This is called mutation. When we changed strings, we actually created a new string formed of the old string plus the changes. With mutations we can modify the original list itself.

List Operators

There are multiple functions we can use to work with lists. Append allows us to tack elements onto the end of a list. Plus concatenates two lists which, unlike append, creates a new list rather than mutating the original lists. Len or length counts the number of elements in a list.

For Loops

We can use for loops to run a function on every element in a list, regardless of how long the list is. This is how we use For:

Session 12

More Object Oriented Programming

In this lesson we explored some topics related to classes and objects. The first new concept was class variables. Previously we have used instance variables, which were unique to each new object of the Movie class. Class variables, however, are variables that all instances share. The example here is the "valid_ratings" variable.

"VALID_RATINGS" = ["G", "PG", "PG-13", "R"]"

This will allow each of our Movie instances to check their ratings against the list of valid ones, to make sure their rating is valid. A variable used in this fashion, is called a constant. This means the values are not likely to change anytime soon. Google's style guide suggests using all caps for variables with a constant value.

Some class variables are predefined in python. This includes __doc__ which allows us to read and write documentation for the class we're using, __name__, which displays the name of the class, and __module__ which displays the name of the module the class is defined in.

This lesson introduced the new concept of inheritance. Well, not brand new. We first discussed the concept of inheritance when dealing with CSS. There it meant that certain values were passed from elements higher on the tree structure to those nested within. So if we set the font for the body of an html document, all the individual paragraphs in that body would use that font by default. Here it's a way to create parent and child classes that share instance variables. By entering the desired parent directory in the normally empy () after our class name. This sets up the relationship between parent and child. We can access the parent information by calling the "Parent.__init__" function from within the child class definition. We concluded this lesson by learning about method overriding where we can change the implementation of a funcion inherited by a child class. We do this by redefining the function in the child class. Now when you call the function from the parent it will behave the way it always did, yet the child will use it's version of the function.

So that's object oriented programming in a nutshell. I had heard the term a million times yet never knew what it meant. Now I do. Not bad!

VOCABULARY

Object: According to wikipedia "In computer science, an object is a location in memory havinga value and possibly referenced by an identifier. An object can be a variable, a data structure, or a function. In the class-based object-oriented programming paradigm, "object" refers to a particular instance of a class where the object can be a combination of variables, functions, and data structures" Wikipedia Object Entry

Class:The template or blueprint used to create specific objects of a similar nature.

Instance: Specific example of any object

Module: A module is a file containing Python definitions and statements. Definitions from a module can be imported into other modules.

Method: A function within a class

Library: A collection of built in functions and modules. The Python Standard Library lets all users of Python access these resources for their programs.

Inheritance:Inheritance allows objects to take on the properties of pre existing objects. Classes can also inherit methods and values from parent classes.

Why OOP is important for large projects

Object oriented programming is useful for larger projects.By using classes we are using a template to create similar programming objects in the future. It is another way we can avoid writing duplicate code, and we know avoiding repetition is always preferable.It also allows us to change things in our code more easily. By using classes to create similar objects, we can make changes to the class and those changes will be made to all the related objects.In the future if a change presents a problem, we will only have to look in one location for the bug rather than in each individual object.

Abstract thinking and connection to other discipline

Object oriented program shares some concepts in common with HTML and CSS.The use of classes is similar. In OOP classes are a template from which objects are created. In CSS we use classes to group together similar elements in a webpage and style them in a uniform fashion. A similarity with HTML is the use of external resources. In OOP we can import definitions and functions from exterior module. In HTML we can embed image files in our page, or even link to other pages. Inheritance is also shared between these disciplines. Objects inherit properties from classes which in turn can inherit properties from super classes. In CSS elements nested in other elements inherit values from their parents. If we set a style for the body element of an HTML file, all the elements within the body will share those styles by default. We also learned more about "abstraction" in which complicated details are hidden so that we can use other peoples code for our projects without having to understand how everything works under the hood. As long as we read the documentation we are able to utilize the code. This means we can focuse on what we want to accomplish with our code without having to reinvent the wheel over and over again.
